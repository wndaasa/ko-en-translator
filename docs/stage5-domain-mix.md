# Stage 5: 도메인 혼합 학습 — "작은 모델이 여러 레지스터를 망각 없이?" (양성)

> 질문: Stage 2에서 학술 파인튜닝이 대화를 망각시켰다. 작은 모델에 일상+격식을 **함께** 담으면
> 또 망가질까(용량 간섭)? **결과: 아니오.** 도메인 태그 혼합 학습 시 일상체는 단일도메인보다
> 오히려 더 잘 됐고(positive transfer), 격식체도 빠르게 학습됨. 60M 모델이 두 레지스터를 동시 보유.

## 데이터 (보유 코퍼스로 제작, 다운로드 없음)

`src/prepare_mixed.py`로 두 레지스터를 균형 혼합 + 도메인 라벨:
- `casual` = OPUS-100 (자막·일상) 70만
- `formal` = AI Hub 기술과학 (학술·격식 문어체) 70만
- 형식: `domain<TAB>ko<TAB>en`, 총 train 140만 / val 2천(각 1천)

> formal은 *학술* 격식체라 공문서/제안서와 100% 같진 않지만, 격식 문어체를 공유하는 프록시.
> "일상 vs 격식 한 모델 공존" 질문 검증엔 충분.

## 방법

- **도메인 태그**: 토크나이저에 `<casual>`/`<formal>` 특수토큰 추가. 소스 = `[방향태그][도메인태그] + 문장`.
- **모델**: minRNN(Stage 4의 효율 구조) 60.5M, 양방향. train.py `--mixed`.
- **핵심 측정**: 검증을 **도메인별로** 나눠 ko→en BLEU를 따로 계산(`evaluate`가 라벨별 BLEU 반환)
  → 간섭/망각을 직접 관찰.

## 결과 (도메인별 BLEU, 동일 학습)

| step | val_loss | formal BLEU | casual BLEU |
|---|---|---|---|
| 2,000 | 5.42 | 3.18 | 0.61 |
| 4,000 | 3.92 | 22.58 | 1.31 |
| 6,000 | 3.20 | 31.20 | 3.48 |
| 8,000 | 2.90 | 35.25 | **3.87** |

**망각/간섭 없음 — 오히려 positive transfer.** 같은 step에서 casual BLEU 비교:

| 모델 (step 8000) | casual BLEU |
|---|---|
| Transformer, OPUS만 | 1.65 |
| minRNN, OPUS만 | 2.87 |
| **minRNN, 혼합(+formal)** | **3.87** |

→ **격식 데이터를 추가했는데 일상체가 떨어지긴커녕 더 좋아짐.** 작은 모델이라 망가질 거란 우려는
이 설정에서 반증됨. formal이 casual보다 빨리/높이 오르는 건 간섭이 아니라 **학술 격식체가 노이즈
적고 정형적이라 본질적으로 학습이 쉬워서**(둘 다 상승 중).

## 해석

- **catastrophic forgetting(순차 파인튜닝 문제)** ≠ **용량 간섭(작은 모델 문제)**. 혼합 학습은
  전자를 제거하고, 후자는 번역 도메인이 구조를 공유해 **약하게만** 나타남(여기선 일상체엔 이득).
- **도메인 태그**가 레지스터를 분리해 한 모델이 둘 다 서비스하게 함.
- 사장님 우려("작아서 또?")에 대한 답: **데이터 혼합 + 태그로 충분히 공존 가능.** 정 부족하면
  도메인 어댑터로 확장(미실행, 후속).
