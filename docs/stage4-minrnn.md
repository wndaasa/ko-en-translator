# Stage 4: minRNN — 선형 순환 seq2seq (양성 결과)

> "노트북에서도 학습·추론이 가벼운 구조"라는 목표에서 출발. 어텐션의 O(n²)·KV 캐시 부담을
> 선형 순환(minGRU)으로 대체하는 가설. **결과: 동일 파라미터에서 품질은 동일 step 기준 트랜스포머를
> 앞서고, 시퀀스가 길어질수록 메모리·속도가 크게 유리.** (Stage 3 MCE의 음성 결과와 대조)

## 동기

트랜스포머의 비용은 모델 크기가 아니라 **(1) 셀프어텐션 O(n²), (2) 자기회귀 디코딩의 KV 캐시,
(3) 순차 생성**에서 온다. 짧은 문장은 작은 트랜스포머도 가볍지만, 긴 문서·온디바이스에선 이게 병목.

**가설**: 게이트를 입력에만 의존시키는 **minGRU**(Feng et al., 2024)로 어텐션을 대체하면,
점화식이 선형이 되어 **학습은 병렬 스캔, 추론은 O(1) 순환**이 된다 → 길이에 선형(O(n)) 비용.

## 설계

minGRU: `h_t = (1−z_t)⊙h_{t-1} + z_t⊙g(h̃_t)`, `z_t=σ(W_z x_t)`, `h̃_t=W_h x_t`.
- `h_t = a_t⊙h_{t-1} + b_t` 형태 → **로그공간 병렬 스캔**(Heinsen, `logcumsumexp`)으로 수치 안정 학습.
- 추론은 `step()`으로 상태만 갱신(O(1) 메모리, KV 캐시 없음).
- seq2seq: 인코더 = **양방향 minGRU**, 디코더 = **causal minGRU + 크로스어텐션**(크로스어텐션은 유지),
  FFN 유지, 위치 인코딩 없음(순환이 순서 내재). 트랜스포머와 동일 인터페이스라 기존 학습/번역 코드 재사용.
- **파라미터 매칭**: minRNN **60,521,472** vs 트랜스포머 **60,524,544** (거의 동일) → "같은 크기, 다른 믹싱"의 공정 비교.

## 검증 (구조 정확성)

`tests/test_minrnn.py` 전부 통과:
- **병렬 스캔 == 순차 계산** (최대오차 1.6e-6) → 학습(병렬)과 추론(순환) 결과 동일 보장
- 디코더 causal 누출 0.0, 양방향 패딩 불변 0.0, weight tying
- 토이 overfit: ko→en BLEU **100** (학습 경로 정상)

## 품질 — 동일 step 비교 (공정)

베이스라인과 **동일 조건**(OPUS 양방향, batch 96, d_model 512, 6+6층, lr 5e-4). 학습 budget을
통제한 **같은 step에서의 검증 지표** 비교 (minRNN은 세션 중단으로 ~1.4 epoch까지; matched-step이라 공정):

| step | val_loss (base / **minRNN**) | ko→en BLEU (base / **minRNN**) |
|---|---|---|
| 2,000  | 5.83 / 5.78 | 0.45 / 0.48 |
| 8,000  | 4.87 / 4.65 | 1.65 / 2.87 |
| 12,000 | 4.62 / 4.39 | 1.89 / 3.24 |
| 16,000 | 4.47 / 4.26 | 3.88 / 5.42 |
| 18,000 | 4.40 / 4.19 | 4.54 / 5.21 |
| 20,000 | 4.36 / **4.15** | 4.43 / **5.54** |

- **10번의 eval 내내 minRNN이 val_loss·BLEU 모두 앞섬.** 동일 파라미터·동일 step에서 더 좋음
  → 이 과제에서 선형 순환이 어텐션보다 **샘플 효율적**(적어도 열세 아님).

## 효율 — 디코딩 길이 스케일링 (핵심)

`tests/benchmark_efficiency.py`, 디코더 forward의 peak 메모리·시간 (d_model 512, 6층, src=50):

| L | peak mem MB (Transformer / **minRNN**) | time ms (Transformer / **minRNN**) |
|---|---|---|
| 128  | 580.7 / 580.9 | 15.2 / 5.4 |
| 1024 | 660.9 / 602.9 | 32.0 / 28.7 |
| 2048 | 879.9 / 627.9 | 89.7 / 55.5 |
| 4096 | **1725.9 / 677.9** | **225.7 / 104.5** |

- 트랜스포머 메모리는 L=128→4096에서 **3배(581→1726MB)로 O(L²) 폭증**, minRNN은 **~1.17배(581→678MB)로 거의 평탄**.
- L=4096에서 minRNN이 **메모리 ~2.5배 적고, 속도 ~2.2배 빠름** — 길수록 격차 확대.
- 이는 forward(병렬) 기준이며, **추론 시 순환 step으로 O(1) 메모리**까지 가능(추가 이득).

## 결론

가설 **지지**. minRNN은 이 번역 과제에서:
1. **동일 파라미터로 품질이 동일 step 기준 트랜스포머를 앞서고**,
2. **시퀀스가 길어질수록 메모리·속도가 크게 유리**(O(L) vs O(L²)) — "가볍게 학습·추론" 목표에 부합.

Stage 3(MCE)는 음성, Stage 4(minRNN)는 양성 — 두 가설을 같은 베이스라인·같은 잣대로 공정 비교한 게 핵심.

## 정직한 한계

- minRNN을 **~1.4 epoch까지만** 학습(세션 중단). matched-step 비교는 공정하나, 완주 비교는 후속 과제.
- BLEU 절대값은 아직 낮음(둘 다 초기 학습 구간).
- **학습 step당 속도**는 minRNN ~0.78×(병렬 스캔이 ROCm에서 어텐션보다 약간 느림) — 이득은 **긴 문맥·추론**에서.
- 크로스어텐션은 유지(완전 attention-free 아님). 단일 시드(분산 미측정).
